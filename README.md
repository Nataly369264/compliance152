## Восстановление фич из другой ветки

Паттерн используется когда нужно перенести конкретные файлы из ветки `main`
в рабочую ветку, не делая полный merge.

```bash
# 1. Перенести нужные файлы из main
git checkout main -- src/module/file.py
git checkout main -- config/sources.yaml

# 2. Проверить что попало в staging
git status

# 3. Закоммитить
git add . && git commit -m "restore: <что восстановили>"
```

**Важно:** после переноса проверяй транзитивные зависимости —
восстановленный модуль может импортировать файлы, которых тоже нет в ветке
(например `src/llm/utils.py`, переменные в `src/config.py`).
Проверка: `python -c "from src.api.server import app"` до запуска сервера.
