## Модуль 1: Scanner — Сканер сайтов

**Задача:** Обойти сайт клиента и собрать всё, что относится к обработке ПДн.

### Что сканирует:
- Все страницы сайта (рекурсивный краулинг с ограничением глубины)
- HTML-формы: поля ввода (name, email, phone, address и т.д.)
- Чекбоксы согласия рядом с формами
- Cookie-баннеры и механизмы управления cookie
- Ссылки на Privacy Policy / Политику обработки ПДн
- Внешние скрипты: Google Analytics, Google Fonts, Facebook Pixel, рекапчи
- Meta-теги, headers (CSP, referrer policy)
- HTTPS-статус
- Футер каждой страницы (наличие ссылки на политику)

### Технологии:
- `httpx` — для HTTP-запросов (быстрый async)
- `BeautifulSoup4` — парсинг HTML
- `Playwright` (опционально) — для JS-рендеринга (SPA-сайты)

### Выход:
```python
@dataclass
class ScanResult:
    url: str
    pages: list[PageInfo]           # Все просканированные страницы
    forms: list[FormInfo]           # Все формы с полями
    cookies: list[CookieInfo]       # Обнаруженные cookie
    external_scripts: list[Script]  # Внешние JS/CSS ресурсы
    privacy_policy: PolicyInfo | None  # Найденная политика (URL, текст)
    ssl_status: SSLInfo             # HTTPS-сертификат
    cookie_banner: BannerInfo | None   # Cookie-баннер
```

## Модуль 2: Analyzer — Анализатор соответствия

**Задача:** На основе ScanResult + база знаний 152-ФЗ сформировать отчёт о нарушениях.

### Чеклист проверок (~40 пунктов):

**Формы (каждая):**
1. Есть ли чекбокс согласия на обработку ПДн
2. Чекбокс по умолчанию не отмечен
3. Есть ли ссылка на Политику обработки ПДн
4. Кнопка submit отключена до согласия
5. Согласие — отдельный документ (не часть пользовательского соглашения)
6. Для маркетинговых рассылок — отдельный чекбокс
7. Форма собирает только необходимый минимум данных
8. В тексте согласия указан оператор (название, ИНН)

**Cookie:**
9. Cookie-баннер появляется при первом визите
10. Есть кнопка "Отклонить" (равноценная "Принять")
11. Аналитические/маркетинговые cookie по умолчанию выключены
12. Есть ссылка на Cookie Policy
13. Нет загрузки аналитики ДО получения согласия
14. Предпочтения cookie сохраняются

**Политика обработки ПДн:**
15. Политика опубликована и доступна без регистрации
16. Ссылка в футере КАЖДОЙ страницы
17. Указано полное наименование оператора
18. Указан ИНН/ОГРН и адрес
19. Указан ответственный за обработку ПДн
20. Перечислены категории собираемых данных
21. Указаны цели обработки для КАЖДОЙ категории
22. Указаны правовые основания
23. Указаны сроки хранения
24. Описаны права субъектов ПДн
25. Описан порядок реализации прав (контакт, сроки — 10 рабочих дней)
26. Информация о трансграничной передаче (если есть)
27. Информация о мерах безопасности
28. Информация о cookies
29. Заявление о локализации данных (хранение в РФ)
30. Дата публикации и последнего обновления
31. Документ на русском языке

**Технические:**
32. HTTPS на всех страницах
33. Нет Google Fonts с серверов Google (замена на локальные)
34. Нет Google Analytics (замена на Яндекс.Метрику)
35. Нет Facebook Pixel
36. Нет Google reCAPTCHA (замена на Яндекс SmartCaptcha или аналог)
37. Нет Google Tag Manager
38. Хостинг на территории РФ

**Регуляторные:**
39. Оператор зарегистрирован в реестре Роскомнадзора
40. Уведомление об обработке ПДн подано

### Анализ через LLM:
- Текст Политики обработки ПДн отправляется в Claude для глубокого юридического анализа
- Claude проверяет полноту, корректность формулировок, соответствие актуальной редакции закона
- Формирует список конкретных замечаний с цитатами из закона

### Выход:
```python
@dataclass
class ComplianceReport:
    site_url: str
    scan_date: datetime
    overall_score: int              # 0-100
    risk_level: str                 # "critical" | "high" | "medium" | "low"
    violations: list[Violation]     # Список нарушений
    recommendations: list[Recommendation]  # Рекомендации
    estimated_fines: FineEstimate   # Оценка возможных штрафов
    checklist: list[CheckItem]      # Полный чеклист с статусами
```

## Модуль 3: Generator — Генератор документов

**Задача:** Сгенерировать полный пакет документов по 152-ФЗ на основе данных об организации.

### Входные данные от клиента:
```python
@dataclass
class OrganizationData:
    legal_name: str                  # Полное наименование
    short_name: str                  # Краткое наименование
    inn: str                         # ИНН
    ogrn: str                        # ОГРН
    legal_address: str               # Юридический адрес
    actual_address: str              # Фактический адрес
    ceo_name: str                    # ФИО руководителя
    ceo_position: str                # Должность руководителя
    responsible_person: str          # ФИО ответственного за ПДн
    responsible_contact: str         # Контакт ответственного
    website_url: str                 # URL сайта
    email: str                       # Email организации
    phone: str                       # Телефон

    # Специфика обработки
    data_categories: list[str]       # Какие данные собирает
    processing_purposes: list[str]   # Цели обработки
    data_subjects: list[str]         # Категории субъектов (клиенты, сотрудники...)
    third_parties: list[str]         # Кому передаёт данные
    cross_border: bool               # Трансграничная передача
    cross_border_countries: list[str] # Страны (если есть)
    hosting_location: str            # Где хостинг
    info_systems: list[str]          # ИСПДн
```

### Генерируемые документы (полный пакет):

**Для сайта (публичные):**
1. Политика обработки персональных данных
2. Согласие на обработку ПДн (для каждой формы)
3. Cookie Policy
4. Пользовательское соглашение (если нужно)

**Внутренние организационные:**
5. Приказ о назначении ответственного за организацию обработки ПДн
6. Приказ об утверждении Политики обработки ПДн
7. Приказ о допуске к обработке ПДн
8. Положение об обработке и защите ПДн
9. Акт оценки вреда субъектам ПДн
10. Модель угроз безопасности ПДн
11. Акт определения уровня защищённости ИСПДн
12. Инструкция по обработке обращений субъектов ПДн
13. Инструкция по реагированию на инциденты
14. Обязательство о неразглашении ПДн
15. Согласие на обработку ПДн сотрудников (письменная форма)
16. Договор поручения обработки ПДн (шаблон для контрагентов)
17. Форма отзыва согласия
18. Уведомление в Роскомнадзор (черновик)

### Как генерирует:
- **Шаблонная основа** — каждый документ имеет юридически выверенный шаблон в knowledge_base/
- **LLM-персонализация** — Claude адаптирует шаблон под конкретную организацию:
  - Подставляет реквизиты
  - Адаптирует перечни данных и целей под конкретный бизнес
  - Формирует корректные юридические формулировки
  - Проверяет внутреннюю непротиворечивость пакета документов
- **Валидация** — каждый документ проверяется на полноту обязательных элементов

### Выход:
- Документы в формате Markdown + HTML + DOCX (python-docx)
- Каждый документ версионирован (дата, номер версии)
- Метаданные в SQLite

## Модуль 4: Monitor — Мониторинг законодательства

**Задача:** Отслеживать изменения в 152-ФЗ, подзаконных актах, разъяснениях РКН и судебной практике.

### Источники мониторинга:
1. **КонсультантПлюс / Гарант** — текст закона и поправки
2. **Роскомнадзор** (rkn.gov.ru) — разъяснения, новости, приказы
3. **Телеграм-каналы** по теме (b152privacy, privacy_tech и т.д.)
4. **Судебные решения** — Судакт, kad.arbitr.ru
5. **Правительственные акты** — publication.pravo.gov.ru

### Как работает:
- Периодический краулинг источников (cron / celery beat)
- Извлечение новых документов/изменений
- Claude анализирует: что изменилось, как влияет на существующие документы клиентов
- Формирует дайджест изменений

### Выход:
```python
@dataclass
class LegalUpdate:
    source: str
    date: datetime
    title: str
    summary: str                    # Краткое описание
    impact_analysis: str            # Как влияет на документы
    affected_documents: list[str]   # Какие документы нужно обновить
    severity: str                   # "critical" | "important" | "informational"
    effective_date: datetime | None # Дата вступления в силу
```

## Модуль 5: Updater — Автообновление документов

**Задача:** При обнаружении изменений в законодательстве автоматически обновить затронутые документы.

### Как работает:
1. Monitor обнаруживает изменение → LegalUpdate
2. Updater определяет, какие документы каких клиентов затронуты
3. Claude генерирует конкретные правки для каждого документа
4. Создаётся новая версия документа (старая сохраняется)
5. Клиент получает уведомление с описанием изменений и diff-ом
6. (В будущем) Автоматическая публикация на сайт клиента

### Режимы:
- **Авто**: документ обновляется сразу, клиент уведомляется постфактум
- **Подтверждение**: клиент получает черновик изменений и подтверждает

## Модуль 6: LLM-клиент (Claude API)

**Переиспользуем паттерн из familytech** (`src/bazi/llm_client.py`):
- Async httpx-клиент для Anthropic API
- Экспоненциальный backoff при ошибках
- Rate limiting
- Structured output через system/user prompts

### Промпты (по модулям):
- `prompts/scanner_analysis.py` — анализ найденного на сайте
- `prompts/policy_review.py` — проверка Политики ПДн
- `prompts/document_generation.py` — генерация каждого типа документа
- `prompts/legal_update_analysis.py` — анализ изменений в законодательстве
- `prompts/document_update.py` — генерация правок в документы

## API (FastAPI)

### Эндпоинты:

```
POST   /api/v1/scan              # Запустить сканирование сайта
GET    /api/v1/scan/{id}         # Получить результат сканирования
POST   /api/v1/analyze           # Запустить анализ соответствия
GET    /api/v1/report/{id}       # Получить отчёт о соответствии
POST   /api/v1/generate          # Сгенерировать пакет документов
GET    /api/v1/documents/{id}    # Скачать документ
GET    /api/v1/documents/package/{org_id}  # Скачать весь пакет
GET    /api/v1/updates           # Получить последние изменения в законодательстве
POST   /api/v1/organizations     # Создать организацию (клиента)
GET    /api/v1/organizations/{id}/status  # Статус соответствия
```
