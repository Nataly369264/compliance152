# CASES.md — Журнал проблемных кейсов сканера

> **Зачем этот файл:** фиксировать ситуации, когда сканер работает не так, как ожидалось.
> Каждый кейс — это будущий тест-кейс и основание для задачи в Claude Code.

---

## Когда записывать кейс

- Запустила сканер, а результат не совпадает с тем, что видишь на сайте глазами
- Claude Code исправил баг — запиши кейс со статусом «исправлено», чтобы потом проверять регрессию
- Кто-то (пользователь, ты, тестировщик) сообщил о проблеме со сканированием конкретного сайта

---

## Как записать кейс за 2 минуты

1. Скопируй шаблон, вставь в конец раздела «Кейсы»
2. Заполни поля со звёздочкой ★ — это минимум, остальное можно дописать позже
3. Номер — просто следующий по порядку

**Категории** (выбери одну или напиши свою):
`policy-not-found` · `policy-wrong-doc` · `forms-missed` · `banner-missed` · `tracker-missed` · `score-wrong` · `crash` · `other`

---

## Шаблон

```
### CASE-NNN: [домен] — [суть в 5–7 словах]
- ★ **Дата:** ГГГГ-ММ-ДД
- ★ **URL:** https://...
- ★ **Категория:** ...
- ★ **Что произошло:** [что показал сканер — 1–2 предложения]
- ★ **Что ожидалось:** [что должно было быть]
- **Где на сайте:** [URL страницы или место: футер, баннер, попап...]
- **Краулер:** SiteScanner | PlaywrightCrawler | оба | не знаю
- **Причина:** [если понятна, иначе не заполняй — выясним с Claude Code]
- **Статус:** открыт | исправлено в [задача] | не воспроизводится | won't fix (причина)
```

---

## Как использовать с Claude Code

Когда начинаешь сессию по исправлению багов сканера, отправь Claude Code:

```
Прочитай CASES.md. Вот открытые кейсы, которые нужно исследовать.
Начни с CASE-NNN: воспроизведи проблему, найди причину, предложи исправление.
```

После исправления — обнови статус кейса и допиши причину.

## Кейсы

### CASE-001: wildberries.ru — сайт полностью недоступен из-за анти-бота
- ★ **Дата:** 2026-04-04
- ★ **URL:** https://wildberries.ru
- ★ **Категория:** policy-not-found
- ★ **Что произошло:** сканер вернул 0 страниц, политика не найдена, ошибка «illegal status line» с HTTP 498
- ★ **Что ожидалось:** политика существует по адресу `/services/privacypolicy/general-terms-of-use/` (проверено curl)
- **Где на сайте:** отдельная страница, путь `/services/privacypolicy/general-terms-of-use/`
- **Краулер:** SiteScanner (httpx)
- **Причина:** Wildberries возвращает нестандартный HTTP 498 + JS-challenge для всех не-браузерных запросов; httpx получает тело HTML как статусную строку и бросает исключение `illegal status line`
- **PlaywrightCrawler:** частично лучше — `pages_scanned=1`, `errors=[]` (vs 0 страниц и ошибка у httpx). Playwright прошёл HTTP 498, загрузил JS-challenge страницу, но bot-detection WB сработал и до реального сайта добраться не удалось. `pp.found=False`, 0 форм, 2 скрипта (скрипты JS-challenge)
- **Статус:** открыт — Playwright не обходит WB anti-bot полностью; нужен реалистичный UA + возможно stealth-режим

### CASE-002: tinkoff.ru — политика найдена, но текст не извлечён (JS-рендеринг)
- ★ **Дата:** 2026-04-04
- ★ **URL:** https://tinkoff.ru
- ★ **Категория:** score-wrong
- ★ **Что произошло:** `privacy_policy.found=True`, URL верный (`/privacy/`), но текст политики — HTML-заглушка «Включите JavaScript»; все поля анализа содержания (`has_data_categories`, `has_purposes`, `has_subject_rights`, `is_russian`) = False
- ★ **Что ожидалось:** текст политики извлечён, поля анализа заполнены корректно
- **Где на сайте:** https://tinkoff.ru/privacy/ — страница требует JS для отображения контента
- **Краулер:** SiteScanner (httpx)
- **Причина:** страница политики рендерится через JavaScript; httpx получает пустой HTML без контента; `is_russian=False` → score за содержание политики = 0 несмотря на наличие ссылки
- **PlaywrightCrawler (до смены UA):** тот же текст-заглушка (`text_len=6722`, `is_russian=False`). Детектирует бота по UA `Compliance152Bot/0.1`.
- **PlaywrightCrawler (после смены UA на Chrome/120):** результат идентичен — `text_len=6722`, `is_russian=False`. Смена UA не помогла. Причина: Tinkoff детектирует headless-режим через `navigator.webdriver=true` — это стандартное свойство Playwright, не зависящее от UA. Нужен stealth-режим: запуск с `--disable-blink-features=AutomationControlled` + патч `navigator.webdriver`.
- **Статус:** открыт — требуется stealth-режим Playwright, не просто смена UA

### CASE-003: ozon.ru — HTTP 403, страница засчитана как просканированная без ошибки
- ★ **Дата:** 2026-04-04
- ★ **URL:** https://ozon.ru
- ★ **Категория:** policy-not-found
- ★ **Что произошло:** сканер вернул `pages_scanned=1`, `errors=[]`, политика не найдена; фактически сайт вернул HTTP 403 с JS-challenge страницей («An...») для всех путей включая `/t/privacy-notice`, `/legal/privacy`, `/privacy`
- ★ **Что ожидалось:** ошибка доступа должна быть зафиксирована; `pages_scanned` не должен засчитывать 403-страницы как успешно просканированные
- **Где на сайте:** https://ozon.ru — 403 Forbidden на всём домене при запросах без browser fingerprint
- **Краулер:** SiteScanner (httpx)
- **Причина:** Ozon использует bot-protection (403 + Cloudflare/собственный WAF); httpx не отдаёт нужный fingerprint браузера; 403-ответ засчитывается сканером как «страница просканирована» без записи в `errors[]`
- **PlaywrightCrawler:** прошёл 403-барьер (`pages_scanned=2`, `errors=[]`), но обнаружил **ложноположительную политику**: `pp.found=True`, `pp.url=https://www.ozon.ru/complaint/support/?incident_id=fab_chlg_...` — это страница bot-challenge, не политика конфиденциальности. `text_len=211`, `is_russian=False`. Playwright справился с HTTP-барьером, но создал новую проблему: ложный `pp.found=True` хуже, чем честный `pp.found=False`
- **Отдельный баг (валидация политики):** `is_privacy_policy_page()` или механизм выбора кандидата принял challenge-URL за политику. Нужна валидация: найденная «политика» должна содержать признаки реального документа (минимальная длина текста, наличие ключевых слов 152-ФЗ, `is_russian=True`), иначе `pp.found` должен оставаться `False`.
- **Статус:** открыт — три независимых бага: (1) HTTP 403 не в errors для SiteScanner; (2) ложный `pp.found=True` при детектировании challenge-страниц; (3) нет валидации качества найденного документа политики

### CASE-004: mos.ru — ConnectTimeout через системный прокси, пустое сообщение об ошибке
- ★ **Дата:** 2026-04-04
- ★ **URL:** https://mos.ru
- ★ **Категория:** crash
- ★ **Что произошло:** сканер вернул `pages_scanned=0`, `errors=['https://mos.ru: ']` (пустое сообщение); политика не найдена
- ★ **Что ожидалось:** либо успешное сканирование, либо ошибка с читаемым сообщением (`ConnectTimeout` или `ProxyTimeout`)
- **Где на сайте:** главная страница — до неё не добрались
- **Краулер:** SiteScanner (httpx)
- **Причина:** httpx с `trust_env=True` (по умолчанию) подхватывает системный HTTP-прокси; TLS-handshake через прокси завершается `ConnectTimeout`; `str(ConnectTimeout)` возвращает `''` → в `errors[]` записывается строка вида `"url: "` без описания типа исключения
- **PlaywrightCrawler:** таймаут 30s на главной (`pages_scanned=0`, ошибка в `errors[]`). Причина та же — системный прокси на тестовой машине, не баг сканера. Playwright честнее фиксирует ошибку в `errors[]`, но результат тот же — сайт недоступен.
- **Статус:** CASE условно закрыт для mos.ru (сетевая проблема окружения, не баг). Баги в сканере: (1) `str(ConnectTimeout)=''` — ошибка логируется без типа исключения (актуально); (2) httpx `trust_env=True` подхватывает системный прокси (актуально для всех окружений с прокси)

### CASE-005: dns-shop.ru — HTTP 401 (Qrator WAF), страница засчитана без ошибки
- ★ **Дата:** 2026-04-04
- ★ **URL:** https://dns-shop.ru
- ★ **Категория:** policy-not-found
- ★ **Что произошло:** сканер вернул `pages_scanned=1`, `errors=[]`, политика не найдена; фактически сайт вернул HTTP 401 с Qrator WAF JS-challenge на всех путях включая `/privacy`, `/info/privacy`, `/about/privacy`
- ★ **Что ожидалось:** политика существует (DNS Shop — крупный ритейл); ошибка доступа должна фиксироваться в `errors[]`
- **Где на сайте:** весь домен — HTTP 401 Qrator WAF
- **Краулер:** SiteScanner (httpx)
- **Причина:** dns-shop.ru использует Qrator WAF (HTTP 401 + JS-challenge `/__qrator/qauth_utm_v2d_v9118.js`); httpx не проходит challenge; 401-ответ засчитывается как «просканированная страница» без записи в errors
- **PlaywrightCrawler:** хуже по результату, но честнее по отчётности — `pages_scanned=0`, `errors=['Page.goto: Timeout 30000ms exceeded']`. Qrator держит headless Chromium в JS-challenge бесконечно, браузер не получает ответа. Плюс: ошибка зафиксирована в `errors[]` (в отличие от SiteScanner). Минус: полный таймаут 30s зря потрачен на главную страницу
- **Статус:** открыт — Qrator блокирует оба краулера; требуется stealth-режим Playwright или реальный браузерный UA + fingerprint

### CASE-006: (конфигурация) — LLM-клиент использует openrouter/free вместо gemini-2.5-pro
- ★ **Дата:** 2026-04-06
- ★ **URL:** n/a (проблема конфигурации, не конкретного сайта)
- ★ **Категория:** other
- ★ **Что произошло:** при запуске полного прогона (`tools/run_golden_scan.py`) LLM-клиент сообщил `model: openrouter/free` вместо ожидаемого `google/gemini-2.5-pro`; вызов прошёл успешно, ответ получен, но модель не соответствует конфигу
- ★ **Что ожидалось:** `OPENROUTER_MODEL=google/gemini-2.5-pro` из `.env` должен передаваться в запрос к OpenRouter API как имя модели
- **Краулер:** не применимо (проблема в `src/llm/client.py`)
- **Причина:** неизвестна; возможные варианты: (1) `OPENROUTER_MODEL` не подхватывается из `.env` при запуске скриптом напрямую; (2) OpenRouter fallback на free-tier при несоответствии ключа и модели; (3) `src/llm/client.py` игнорирует конфиг и хардкодит модель
- **Статус:** открыт — требует отдельной диагностики в следующей сессии

### CASE-007: `_select_best_policy` — преждевременная фильтрация `max()` до валидации
- ★ **Дата:** 2026-04-10
- ★ **URL:** n/a (архитектурная находка)
- ★ **Категория:** bug
- ★ **Что произошло:** `max()` в `_select_best_policy` выбирает одного победителя по длине текста, и только после этого валидирует его через `is_valid_policy_text`. Если победитель не прошёл валидацию — возвращается пустой `PrivacyPolicyInfo`, остальные кандидаты не рассматриваются. Это означает, что одна «длинная» страница-самозванец может перебить настоящую политику в выборе.
- ★ **Что ожидалось:** валидировать всех кандидатов, выбирать лучшего ИЗ прошедших валидацию
- **Краулер:** оба
- **Статус:** открыт — обнаружен при диагностике сессии 1, не блокирует el-ed.ru, к починке позже на сайтах с несколькими кандидатами в политики
